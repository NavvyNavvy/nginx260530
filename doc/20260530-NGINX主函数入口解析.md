## 主要学习内容

### 初始化阶段（206-234行）

```c
ngx_debug_init();                      // 初始化调试系统

if (ngx_strerror_init() != NGX_OK) {  // 初始化 strerror 错误码字符串表
    return 1;
}

if (ngx_get_options(argc, argv) != NGX_OK) {  // 解析命令行参数 -v -h -t -c 等
    return 1;
}

ngx_time_init();                       // 初始化时间模块
ngx_regex_init();                      // 初始化正则表达式（如果启用PCRE）

ngx_pid = ngx_getpid();                // 记录当前进程ID
ngx_parent = ngx_getppid();            // 记录父进程ID
```

**学习点**：
- `ngx_get_options()` 是经典的命令行参数解析 — 手动解析 `argv`，处理 `-p`, `-c`, `-s` 等选项
- 注意错误处理模式：每个初始化函数都返回值 `ngx_int_t`，失败返回 `NGX_ERROR`（通常为-1）
- 时间、正则等子系统在最早阶段就需要初始化，因为后续模块依赖它们

### 日志系统初始化（235-253行）

```c
log = ngx_log_init(ngx_prefix, ngx_error_log);
if (log == NULL) {
    return 1;
}

ngx_memzero(&init_cycle, sizeof(ngx_cycle_t));  // 清零结构体
init_cycle.log = log;
ngx_cycle = &init_cycle;                         // 全局指针指向init_cycle

init_cycle.pool = ngx_create_pool(1024, log);   // 创建初始内存池（1KB）
if (init_cycle.pool == NULL) {
    return 1;
}
```

**学习点**：
- `ngx_cycle_t` 是 NGINX 最核心的结构体之一 — 贯穿整个生命周期
- `ngx_create_pool()` 体现 NGINX 的内存管理哲学：**提前分配、统一释放**
- `NGX_CONF_UNSET` 等 sentinel 值用于区分"未设置"和"设置为0"

### OS相关初始化（267-283行）

```c
if (ngx_os_init(log) != NGX_OK) {      // 检测CPU、内存页大小、文件描述符等
    return 1;
}

// 下面两个初始化依赖 ngx_os_init() 设置的参数
if (ngx_crc32_table_init() != NGX_OK) {   // CRC32查表初始化
    return 1;
}

ngx_slab_sizes_init();                    // slab内存分配器大小参数
```

**学习点**：
- `ngx_os_init()` 探测平台能力（CPU数量、cache line大小、系统限制）
- CRC32 使用查表法优化（`ngx_crc32_table_init()` 预计算256项表）
- slab allocator 需要先知道页大小才能计算管理结构大小

### 继承socket和模块预初始化（285-293行）

```c
if (ngx_add_inherited_sockets(&init_cycle) != NGX_OK) {
    return 1;
}

if (ngx_preinit_modules() != NGX_OK) {    // 遍历所有模块，填充名字和index
    return 1;
}
```

**学习点**：
- **热升级**机制：老进程通过环境变量 `NGINX_VAR`（"NGINX"）传递打开的socket给新进程
- `ngx_add_inherited_sockets()` 解析环境变量中的socket FD列表
- `ngx_preinit_modules()` 让所有模块知道自己"在数组中的第几位"

### 创建cycle（293-330行）

```c
cycle = ngx_init_cycle(&init_cycle);   // 核心！解析配置文件，创建listen socket
if (cycle == NULL) {
    if (ngx_test_config) {
        ngx_log_stderr(0, "configuration file %s test failed",
                       init_cycle.conf_file.data);
    }
    return 1;
}

// 配置测试模式：只验证配置文件，不启动
if (ngx_test_config) {
    if (!ngx_quiet_mode) {
        ngx_log_stderr(0, "configuration file %s test is successful",
                       cycle->conf_file.data);
    }
    return 0;
}

// 信号控制模式：nginx -s reload/quit/stop
if (ngx_signal) {
    return ngx_signal_process(cycle, ngx_signal);
}
```

**学习点**：
- `ngx_init_cycle()` 是**最复杂的初始化函数** — 解析配置、创建监听socket、初始化共享内存
- `ngx_cycle_t` 包含 `conf_ctx`（所有模块配置的void*数组）
- 信号处理允许 master 进程接收 `reload`（重载配置）、`quit`（优雅退出）等命令

### 守护进程化（343-361行）

```c
#if !(NGX_WIN32)
if (ngx_init_signals(cycle->log) != NGX_OK) {
    return 1;
}

if (!ngx_inherited && ccf->daemon) {
    if (ngx_daemon(cycle->log) != NGX_OK) {  // fork() + setsid()
        return 1;
    }
    ngx_daemonized = 1;
}

if (ngx_inherited) {
    ngx_daemonized = 1;
}
#endif
```

**学习点**：
- `ngx_daemon()` 实现经典守护进程：`fork()` → 父进程退出 → `setsid()` 创建新会话
- Windows 版本不调用 `ngx_daemon()`（`NGX_WIN32` 宏）
- `ngx_inherited` 标记是否从老进程继承了socket（热升级场景）

### 进入事件循环（380-385行）

```c
if (ngx_process == NGX_PROCESS_SINGLE) {
    ngx_single_process_cycle(cycle);   // 单进程模式（调试用）
} else {
    ngx_master_process_cycle(cycle);   // 多进程 master-worker 模式
}
```

**学习点**：
- NGINX 有三种进程模式：`NGX_PROCESS_SINGLE`（无master）、`NGX_PROCESS_MASTER`、`NGX_PROCESS_SIGNALLER`
- 大部分生产环境用 `ngx_master_process_cycle()` — master 管理 worker，worker 处理请求
